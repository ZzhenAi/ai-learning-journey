# AI迭代追问建议器

帮助用户分析AI回答，给出下一轮追问建议，让输出质量从60分提升到90分。

## 触发条件

✅ **触发此 skill：**

- 「这个回答不够好，怎么追问？」
- 「帮我优化追问」
- 「AI回答不满意，下一步怎么问？」
- 「给我迭代建议」
- 用户粘贴了一段AI回答并说「继续优化」

## 使用前提

用户需要提供：
1. 原始提问（可选，帮助理解上下文）
2. AI 的回答（必填）
3. 用户不满意的地方（可选，让建议更精准）

## 迭代追问的四种方向

### 1. 补充约束

**什么时候用：** 回答方向对，但不够精确、不够具体

**示例追问：**
- 「预算控制在 X 万以内」
- 「字数控制在 X 字以内」
- 「只考虑 XX 品牌」
- 「时间范围限定在 XX」

### 2. 指出问题

**什么时候用：** 回答有遗漏、有错误、有矛盾

**示例追问：**
- 「缺少 XX 部分，补上」
- 「第 X 点的数据有问题，重新计算」
- 「这里的说法和前面矛盾了，统一一下」
- 「没有考虑 XX 情况」

### 3. 调整方向/修改建议

**什么时候用：** 回答整体风格或方向偏了

**示例追问：**
- 「风格偏商务了，加一些温暖感」
- 「太复杂了，简化成甲方能看懂的版本」
- 「换个角度，从 XX 的立场重写」
- 「太正式了，改成口语化表达」

### 4. 深化细节

**什么时候用：** 回答框架对，但细节不够深入

**示例追问：**
- 「第 X 点展开到具体做法」
- 「这个材料给出具体规格和用量」
- 「这个节点画出示意，标注关键尺寸」
- 「每一步加上时间预估」

## 使用步骤

### 第 1 步：分析 AI 回答

识别回答的问题类型：
- 是否缺少约束？
- 是否有遗漏/错误？
- 是否方向偏了？
- 是否细节不够？

### 第 2 步：生成追问建议

针对问题类型，给出 3-5 条追问建议，每条包含：
- 追问内容
- 什么情况下用

### 第 3 步：输出追问模板

给出可以直接复制粘贴的追问语句。

## 输出格式（Word 文档格式）

```
## AI 迭代追问建议

### 原回答问题分析

| 问题类型 | 具体问题 |
|----------|----------|
| 约束不足 | 预算、时间、范围等约束缺失 |
| 内容遗漏 | 缺少 XX 部分 |
| 方向偏差 | 风格偏 XX，需要调整 |
| 细节不够 | 第 X 点需要深化 |

---

### 追问建议

**建议 1：补充约束**

- 追问内容：「预算控制在 X 万以内，重新调整方案」
- 适用情况：回答没有考虑预算限制

**建议 2：指出问题**

- 追问内容：「缺少 XX 部分，补上」
- 适用情况：回答有遗漏

**建议 3：调整方向**

- 追问内容：「风格偏 XX 了，改成 XX 方向」
- 适用情况：回答风格或方向不对

**建议 4：深化细节**

- 追问内容：「第 X 点展开到具体做法和用量」
- 适用情况：回答太笼统，需要具体化

---

### 直接可用的追问语句

1. 「预算控制在 X 万以内，重新调整方案」
2. 「缺少 XX 部分，补上」
3. 「风格偏 XX 了，改成 XX 方向」
4. 「第 X 点展开到具体做法和用量」

---

**建议追问顺序：** 先补充约束 → 再指出问题 → 再调整方向 → 最后深化细节
```

## 迭代成功率经验

| 迭代轮数 | 分数提升 | 关键动作 |
|----------|----------|----------|
| 第 1 轮 | 60 → 75 | 确认框架、补充约束 |
| 第 2 轮 | 75 → 85 | 指出遗漏、深化细节 |
| 第 3 轮 | 85 → 90+ | 打磨细节、调整风格 |

## 推荐模型

- **GLM**：对话体验好，迭代建议自然流畅
- **DeepSeek**：逻辑分析强，能精准定位问题

## 使用技巧

1. **不要重新开对话** — 在同一个对话里迭代，AI 会记住上下文
2. **一轮只改一个方向** — 别一次提太多要求，容易乱
3. **2-3 轮到 85 分就够** — 不要追求完美，差不多就能用
4. **记录好的追问** — 下次遇到类似情况直接用

## 注意事项

- 追问要具体，不要说「再改改」这种模糊的话
- 每轮迭代后，检查是否偏离原目标
- 如果迭代 3 轮还是不满意，可能是原始提问就有问题，重新设计提问
